# Support

Use GitHub issues for reproducible project problems and scoped feature requests.

## Where To Ask

- Bug reports: open a bug report issue.
- Feature ideas: open a feature request issue.
- Security issues: follow `SECURITY.md`.
- General usage questions: open a discussion if GitHub Discussions are enabled; otherwise open an issue with the `question` label.

## What To Include

- Slavey version or commit.
- Operating system and architecture.
- Whether you are running `npm run dev`, `npm run dev:web`, or a packaged app.
- Relevant validation output from `npm run check` or targeted tests.
- A minimal reproduction when possible.

Please do not include secrets, credentials, private prompts, raw terminal output, environment variables, or private repository contents in public reports.

## Support Expectations

This is an open source project, not a paid support channel. Maintainers may close issues that are out of scope, inactive, not reproducible, or missing enough information to act on.
