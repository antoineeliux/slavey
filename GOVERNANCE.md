# Governance

Slavey is maintainer-led. The maintainer sets project direction, reviews contributions, and decides what is accepted into the repository.

## Maintainer

- Primary maintainer: Antoine Liu (`@antoineeliux`)

## Decision Making

Routine fixes and small improvements are handled through issues and pull requests. Larger changes should start with an issue that explains the problem, tradeoffs, user impact, and testing plan before implementation.

Maintainers may decline changes that are useful but outside the current project scope, too risky for local workspace safety, insufficiently tested, or inconsistent with the architecture.

## Releases

Releases are cut by maintainers. Pre-1.0 releases may include breaking changes when needed, but changes should be documented in `CHANGELOG.md` and release notes.
